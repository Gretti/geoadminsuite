1) Récupérer les données Frida sur :  ftp://ftp.intevation.de/freegis/frida/frida-1.0.1-shp.tar.gz
     (infos complémentaires: http://frida.intevation.org/)
    
2) Créer une base de données spatiales 
    (encoding UTF8) nommée: frida

    Vérifier que la base de données est bien spatiale

3) Charger les données dans la base frida. 
    • le nom de la table a creer est donné entre parentheses, 
    • le code EPSG des données est: 31467

  gewaesserflaechen.shp          	-> 	lacs (lake)
  gewaesserlinien-joined.shp		-> 	rivières et canaux (river)
  strassen-joined.shp				-> 	rues (street)

  Vérifier que les données sont bien:
  - Topologiquement valides
  - Du bon type géométrique
  - Du bon système de projection en base

4) Ecrire les requêtes spatiales suivantes:
  a) Récupérer les noms des deux lacs ayant la surface la plus importante
  b) Créer un buffer de 200 mètres autour du plus petit lac, et calculer l'aire du buffer obtenu
  c) Calculer la distance en mètre (avec arrondi) entre les centroid du plus petit et du plus grand lac
  d) Récupérer les identifiants (strid) de toutes les segments de rues
      dont la géométrie intersecte un buffer de 200m autour du plus petit lac
  e) Aggréger les géométries (via ST_Union) récupérées lors de la requête précedente, 
      ayant le même identifiant (strid) 

5) Visualisation et export
  a) Placer les résultats précédent (en e) dans une nouvelle table nommée 'rue_parc' 
      (mettre à jour geometry_columns, et créer un index)
  b) Visualiser les couches avec Udig, GvSIG ou QGIS
  c) Créer un export Shapefile de la table 'rues' en reprojetant en latitude/longitude (EPSG:4326)
